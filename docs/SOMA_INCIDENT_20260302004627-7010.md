## Root Cause Analysis (Code-Based)

### Failure Location

- **File:** `ops/scripts/soma_run_to_done.py`
- **Function:** `main()` (TRIGGER block, lines 191–203)
- **Condition:** `code not in (200, 202)` after `_curl("POST", "/api/exec", data={"action": "soma_kajabi_auto_finish"}, timeout=5)`

### Classification

**HOST_EXECUTOR** — trigger-layer failure: `soma_run_to_done` could not successfully dispatch `soma_kajabi_auto_finish` via HQ. The Soma action itself never ran.

### Pre-check vs Wrapped Downstream

**Pre-check failure.** The TRIGGER step is a pre-check: it verifies that HQ accepts the `soma_kajabi_auto_finish` request before polling. If the POST fails, `soma_kajabi_auto_finish` is never started.

### Likely Causes (from code paths)

| Cause | HTTP Code | Scenario |
|-------|-----------|----------|
| **Timeout** | -1 (urllib) | `timeout=5` on TRIGGER POST (line 191). HQ `apps/openclaw-console/src/app/api/exec/route.ts` lines 431–440: `checkConnectivity` with backoff [10s, 20s, 40s] before 502. If hostd /health is slow, soma_run_to_done times out before HQ responds. |
| ALREADY_RUNNING | 409 | `soma_kajabi_auto_finish` lock held; another run in progress. |
| HOSTD_UNREACHABLE | 502 | HQ could not reach hostd; returns 502 after probe. |
| ORIGIN_BLOCKED | 403 | CSRF rejection (less likely for localhost curl). |
| Other | 4xx/5xx | Schema, maintenance, etc. |

**Primary hypothesis:** 5s timeout is too short for HQ’s hostd probe; connection/timeout leads to `code=-1`, `body` containing timeout message, and TRIGGER_FAILED.
